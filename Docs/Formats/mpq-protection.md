# MPQ Protection Techniques

StarCraft 맵 보호에 사용되는 MPQ 레벨 기법과 이 도구의 대응.

## 1. 해시 인덱스 변조

**방법**: 해시 테이블의 `staredit\scenario.chk` 엔트리 인덱스를 잘못된 값으로 설정.
**대응**: `PatchProtectedHashIndexes` — 올바른 해시값 계산 후 인덱스 보정.

## 2. 해시/블록 테이블 오프셋 변조

**방법**: MPQ 헤더의 테이블 오프셋을 의도적으로 잘못된 위치로 설정.
**대응**: `RecoverShiftedProtectedTables` — 파일 내에서 실제 테이블 위치를 탐색하여 복구.

## 3. MPQ 헤더 손상

**방법**: 헤더의 해시/블록 카운트를 비정상적으로 크게 설정하여 MPQ 라이브러리 크래시 유도.
**대응**: Deep Recovery — TkMPQ 오픈 실패 시, 파일을 직접 스캔하여 MPQ 시그니처(`MPQ\x1A`) 위치를 찾고 테이블 후보를 시도.

## 4. 시나리오 파일 경로 변조

**방법**: `staredit\scenario.chk` 대신 다른 경로로 CHK를 저장.
**대응**: 알려진 경로 목록을 순회하며 시도.

## CHK 레벨 보호

### SMLP 섹션

보호 마커로 사용. ScmDraft가 이 섹션을 보면 열기를 거부함. → 무조건 제거.

### 중복 섹션

동일한 이름의 섹션을 여러 개 삽입. StarCraft는 마지막 것을 사용하지만 에디터는 첫 번째를 읽을 수 있음. → 마지막 것만 유지.

### 가짜 UNIT 레코드

유효하지 않은 유닛 타입이나 좌표를 가진 레코드 삽입. → 필터링.

### 가짜 TRIG 레코드

실행 불가능한 트리거 삽입. → 필터링.

### Freeze05 보호

EUD(Extended Unit Deaths) 트리거를 이용한 고급 보호. 메모리 조작을 통해 맵 실행 시에만 정상 동작하도록 함. → 시드키/목적지키 감지 후 해당 EUD 트리거 제거.
